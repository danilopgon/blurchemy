# Blurchemy Impeccable Context

## Design Context

### Users

Blurchemy serves visual tinkerers, designers, photographers, musicians, privacy-conscious creators, and technical users who want to experiment with image transformations quickly in the browser. Their immediate job is to open the editor, import a local image, understand what is available now, and trust that their image never leaves the browser.

### Brand Personality

Blurchemy should feel focused, tactile, and honest. It is a quiet creative darkroom rather than a SaaS dashboard: precise about technical facts, calm about limitations, and respectful of the user's image privacy.

### Aesthetic Direction

The intended Spec 1 direction is a dark creative workspace with warm-neutral near-blacks, restrained surfaces, a single cyan accent, and a transparency checkerboard treatment that makes the image feel like the primary subject. UI type should be neutral and legible, while technical metadata uses monospace to signal precision. The interface should avoid marketing-page energy, decorative gradients, fake capability, and overbuilt dashboard conventions.

### Design Principles

1. Make the canvas the subject: keep chrome quiet so the imported image carries the screen.
2. Stay honest about scope: never imply effects, presets, export, upload, storage, or processing exist before they do.
3. Protect local trust: every import interaction should reinforce that image data stays in the browser.
4. Treat every state as designed: empty, importing, imported, and error states must be clear, accessible, and specific.
5. Use restraint with precision: cyan, motion, typography, and technical metadata should have explicit jobs, not decorative roles.
